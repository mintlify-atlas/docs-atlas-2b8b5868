---
title: "Login"
description: "Authenticate with email and password. Returns a JWT access token and refresh token."
---

## Endpoint

```
POST /api/v1/auth/login
```

**Authentication:** None required — this endpoint is public (`AllowAnonymous`).

## Request body

<ParamField body="email" type="string" required>
  The registered email address of the user.
</ParamField>

<ParamField body="password" type="string" required>
  The user's password.
</ParamField>

## Response

Returns an `AuthResponse` object on success.

<ResponseField name="accessToken" type="string">
  A signed JWT used to authenticate subsequent API requests. Include it as `Authorization: Bearer <accessToken>`.
</ResponseField>

<ResponseField name="refreshToken" type="string">
  An opaque token used to obtain a new `accessToken` when the current one expires.
</ResponseField>

<ResponseField name="accessTokenExpiresAt" type="string (ISO 8601)">
  UTC timestamp indicating when the `accessToken` expires.
</ResponseField>

<ResponseField name="tokenType" type="string">
  Always `"Bearer"`.
</ResponseField>

## Status codes

| Code | Description |
|---|---|
| `200 OK` | Authentication succeeded. |
| `401 Unauthorized` | Email or password is incorrect. |

## Example

<CodeGroup>
```bash curl
curl -X POST https://api.cobranzadigital.com/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "cashier@tienda.mx",
    "password": "Segura123!"
  }'
```

```json Response
{
  "accessToken": "eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCJ9...",
  "refreshToken": "dGhpcyBpcyBhIHJlZnJlc2ggdG9rZW4...",
  "accessTokenExpiresAt": "2026-03-17T15:00:00Z",
  "tokenType": "Bearer"
}
```
</CodeGroup>

## Error response example

```json
{
  "type": "https://tools.ietf.org/html/rfc9110#section-15.5.1",
  "title": "Invalid credentials",
  "detail": "The email or password is incorrect.",
  "status": 401
}
```

<Note>
The error message intentionally does not reveal whether the email exists. Both "email not found" and "wrong password" return the same `401` response.
</Note>
