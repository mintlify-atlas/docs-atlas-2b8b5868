---
title: "Platform Dashboard"
description: "Cross-tenant KPIs, sales trends, inventory alerts, and operational signals for SuperAdmin operators."
---

<Warning>
  All endpoints on this page enforce the `PlatformOnly` policy (`SuperAdmin` role).
  Do not send `X-Tenant-Id`; it is ignored by all dashboard routes.
</Warning>

The Platform Dashboard API provides a real-time cross-tenant view of the CobranzaDigital estate. It is grouped into three tiers:

- **v1 — Core KPIs**: summary counts, top-tenants, alerts, inventory and recent adjustments
- **v2 — Executive metrics**: sales trend, void leaders, stockout hotspots, activity feed, executive signals
- **v3 — Drill-down**: alert drilldown, per-tenant overview, per-store stockout details

---

## GET /summary

```
GET /api/v1/platform/dashboard/summary
```

Returns a platform-wide snapshot including active/inactive counts, today's sales, open shifts, and low-stock alerts.

### Query parameters

<ParamField query="dateFrom" type="string (ISO 8601)">
  Start of the reporting window. Defaults to the current day in UTC if omitted.
</ParamField>

<ParamField query="dateTo" type="string (ISO 8601)">
  End of the reporting window. Defaults to now in UTC if omitted.
</ParamField>

<ParamField query="threshold" type="number" default="5">
  Low-stock quantity threshold. Items with `onHand ≤ threshold` are counted as low stock.
</ParamField>

### Response `200 OK`

<ResponseField name="activeTenants" type="integer">
  Number of tenants with `isActive = true`.
</ResponseField>

<ResponseField name="inactiveTenants" type="integer">
  Number of tenants with `isActive = false`.
</ResponseField>

<ResponseField name="activeStores" type="integer">
  Total active stores across all tenants.
</ResponseField>

<ResponseField name="inactiveStores" type="integer">
  Total inactive stores across all tenants.
</ResponseField>

<ResponseField name="totalUsers" type="integer">
  Total registered users platform-wide.
</ResponseField>

<ResponseField name="usersWithoutStoreAssignment" type="integer">
  Users who have no store assigned (potential misconfiguration).
</ResponseField>

<ResponseField name="tenantsWithoutCatalogTemplate" type="integer">
  Tenants that have no catalog template linked.
</ResponseField>

<ResponseField name="storesWithoutAdminStore" type="integer">
  Stores with no user holding the `AdminStore` role.
</ResponseField>

<ResponseField name="salesTodayCount" type="integer">
  Number of completed sales on today's calendar date (UTC).
</ResponseField>

<ResponseField name="salesTodayAmount" type="number">
  Total monetary value of today's sales.
</ResponseField>

<ResponseField name="salesLast7DaysCount" type="integer">
  Number of completed sales in the last 7 days.
</ResponseField>

<ResponseField name="salesLast7DaysAmount" type="number">
  Total monetary value of the last 7 days' sales.
</ResponseField>

<ResponseField name="openShiftsCount" type="integer">
  Number of POS shifts currently open.
</ResponseField>

<ResponseField name="outOfStockItemsCount" type="integer">
  Catalog items with zero on-hand inventory across all stores.
</ResponseField>

<ResponseField name="lowStockItemsCount" type="integer">
  Items with `onHand > 0` but `onHand ≤ threshold`.
</ResponseField>

<ResponseField name="effectiveDateFromUtc" type="string (ISO 8601)">
  The resolved start of the reporting window used by the server.
</ResponseField>

<ResponseField name="effectiveDateToUtc" type="string (ISO 8601)">
  The resolved end of the reporting window used by the server.
</ResponseField>

<ResponseField name="effectiveLowStockThreshold" type="number">
  The threshold value actually applied.
</ResponseField>

### Example

## GET /alerts

```
GET /api/v1/platform/dashboard/alerts
```

Returns a categorized list of operational alerts across the platform.

### Alert codes

| Code | Description |
|------|-------------|
| `TENANT_WITHOUT_TEMPLATE` | Tenant has no catalog template assigned. |
| `STORE_WITHOUT_ADMINSTORE` | Store has no user with the `AdminStore` role. |
| `STORE_SCOPED_USER_WITHOUT_STORE` | User scoped to a store but has no store assignment. |

### Response `200 OK`

<ResponseField name="alerts" type="array">
  <Expandable title="PlatformDashboardAlertDto">

    <ResponseField name="severity" type="string">Alert severity level, e.g. `warning` or `error`.</ResponseField>
    <ResponseField name="count" type="integer">Number of entities affected by this alert.</ResponseField>
    <ResponseField name="description" type="string">Human-readable description of the alert.</ResponseField>
    <ResponseField name="topExamples" type="string[]">Up to 3 example entity names affected by the alert.</ResponseField>
  </Expandable>
</ResponseField>

### Example

## GET /alerts/drilldown

```
GET /api/v1/platform/dashboard/alerts/drilldown
```

Returns row-level detail for a specific alert code, optionally filtered to a single tenant or store.

### Query parameters

<ParamField query="code" type="string" required>
  Alert code to drill into. Must be one of `TENANT_WITHOUT_TEMPLATE`, `STORE_WITHOUT_ADMINSTORE`, or `STORE_SCOPED_USER_WITHOUT_STORE`. Returns `400` for unknown codes.
</ParamField>

<ParamField query="take" type="integer" default="100">
  Maximum number of rows to return. Max `500`.
</ParamField>

<ParamField query="tenantId" type="string (uuid)">
  Limit results to a specific tenant.
</ParamField>

<ParamField query="storeId" type="string (uuid)">
  Limit results to a specific store.
</ParamField>

### Response `200 OK`

<ResponseField name="code" type="string">The alert code that was queried.</ResponseField>

<ResponseField name="items" type="array">
  <Expandable title="PlatformDashboardAlertDrilldownItemDto">
    <ResponseField name="tenantId" type="string (uuid) | null">Affected tenant, if applicable.</ResponseField>
    <ResponseField name="tenantName" type="string | null">Tenant display name.</ResponseField>
    <ResponseField name="storeId" type="string (uuid) | null">Affected store, if applicable.</ResponseField>
    <ResponseField name="storeName" type="string | null">Store display name.</ResponseField>
    <ResponseField name="userId" type="string (uuid) | null">Affected user, if applicable.</ResponseField>
    <ResponseField name="userName" type="string | null">User's username.</ResponseField>
    <ResponseField name="email" type="string | null">User's email address.</ResponseField>
    <ResponseField name="role" type="string | null">User's role, if relevant.</ResponseField>
    <ResponseField name="description" type="string">Human-readable explanation of why this row is flagged.</ResponseField>
    <ResponseField name="reason" type="string | null">Additional machine-readable reason code.</ResponseField>
    <ResponseField name="metadata" type="object | null">Arbitrary key/value pairs with extra context.</ResponseField>
  </Expandable>
</ResponseField>

### Status codes

| Code | Description |
|------|-------------|
| `200` | Success (may return `items: []` when no issues found). |
| `400` | `code` is missing or not a recognized alert code. |
| `401` | Missing or invalid token. |
| `403` | Caller is not a SuperAdmin. |
